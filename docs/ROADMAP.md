# DevTycoon 개발 로드맵

MVP 프로토타입(`prototype/devtycoon.html`, 431L 단일 파일 vanilla JS)을 기반으로,
`CLAUDE.md`의 멀티에이전트 하네스를 사용해 React + Vite + TS 제품으로 키워가는 계획.

**제약:** 로컬 전용, 서버·DB 없음, 영속성은 localStorage.

---

## MVP 현황 요약 (무엇이 이미 있나)

| 영역 | MVP 보유 | 비고 |
|---|---|---|
| 자원 | LoC / ₩ / 유저 / 기술부채 / 경력 (5종) | 탄탄한 코어 |
| 액션 | 코드 짜기, 기능 출시, 리팩토링 | 클릭 + 능동 선택 |
| 자동화 | 개발자 6티어 채용 | 비용 ×1.15 증가 |
| 핵심 긴장 | 부채 → 생산성 ↓ (`1/(1+debt/120)`) + 버그 이벤트 | 장르적으로 신선 |
| 환생 | 누적수익→경력, 경력당 전역 +2% | 1개 메타 레이어 |
| 연출 | 글리치, 플로팅 숫자, 커밋 로그 | game-feel 기초 |
| **없음** | **저장/오프라인, 업그레이드, 연구, 이벤트 선택, 도전과제, 사운드, 모바일 최적화** | 확장 여지 |

### 밸런스 빠른 관찰 (정밀 튜닝은 `balance-tuner`가 시뮬레이션으로)
- 첫 환생 문턱: `totalWon ≥ 1e6`에서 경력 +1 → 첫 런이 꽤 길 가능성. 곡선 점검 필요.
- 초반 클릭력 `(1+career)*careerMult`은 경력 0일 때 1 고정 → 업그레이드 없으면 초반 클릭 성장감 약함 → **클릭 강화 업그레이드 우선순위 ↑**.
- 부채 페널티가 LoC·₩ 동시 감산 → 후반 dev 스택 시 리팩토링 압박 큼(의도된 긴장). 리팩토링 비용 `max(50, debt*5)` 스케일 확인.

## 페이즈 계획

각 페이즈는 하네스 파이프라인으로 진행: `game-designer → balance-tuner → planner → react-game-engineer → code-reviewer/react-reviewer (+performance-optimizer) → /playtest`.

### Phase 0 — React+TS 기반 이식 (Foundation) ✅ 완료
MVP 로직을 우리 아키텍처 규칙(`rules/game/architecture.md`)에 맞게 이식.
- Vite + React + TS 스캐폴드, ESLint/Prettier.
- **순수 sim 레이어** `src/game/sim/`: `tick(state, dt)`, `cost`, `production`, `prestige` — React 비의존, 단위테스트 가능.
- 상수/콘텐츠 분리: `src/game/config/balanceConfig.ts`(현 `<script>` 상단 상수), `src/game/content/devs.ts`(현 `DEVS`).
- **단일 rAF 틱 루프** + 델타 클램프 (`idle-game-loop` 스킬).
- Zustand 스토어 + 셀렉터. 라이브 카운터는 리프 컴포넌트로 격리(전체 리렌더 방지).
- 빅넘버 표기 유지(`fmt`) → 추후 `break_infinity` 도입 여지.
- **수용 기준:** MVP와 동일 플레이 가능 + `npm run build` 통과 + sim 단위테스트 그린.

### Phase 1 — 영속성 & 오프라인 (원본 메모 [1]) ✅ 완료
`localstorage-save-system` 스킬 + `rules/game/save-system.md`.
- 버전드 세이브 + 마이그레이션, 디바운스 자동저장, `visibilitychange`/`beforeunload` 저장.
- 오프라인 정산 **최대 8h**(MVP 주석 스펙 유지), 효율 < 1, "자리를 비운 사이…" 요약 모달.
- 세이브 **export/import**(base64 복붙) = 백엔드 없는 "클라우드 세이브".
- 수동 저장 / 하드 리셋(확인 다이얼로그).
- **수용 기준:** 새로고침 후 진행 유지, 8h 캡 정상, 손상 세이브 시 `.bak` 백업 후 복구.

### Phase 2 — 업그레이드 트리 & 밸런싱 (원본 메모 [2][3]) ✅ 완료
- 업그레이드 패밀리 4종(11개): **클릭 강화**(LoC 구매), **생산 ×**, **부채 발생 감소**, **오프라인 효율 ↑**. 1회성 구매, 환생 시 초기화. 효과는 순수 economy 레이어에서 합산.
- 세이브 스키마 v2 + v1→v2 마이그레이션(업그레이드 필드 추가).
- 밸런스 시뮬레이션(`balance.sim.test.ts`, 그리디 오토플레이어)으로 검증:
  - 첫 구매 ~2s, 유휴>클릭 교차 180s, **첫 환생 ~163분**(좋은 플레이 기준), 데드존/런어웨이 없음.
  - **핵심 발견 — 부채 데스스파이럴:** prodMult가 무너지면 LoC·₩가 동시에 죽어 리팩토링 비용을 못 내고 회복 불가. 리팩토링 예비금을 두면 회피됨. (튜닝/UX 후보: 리팩토링 비용 상한, prodMult 하한, 또는 위험 경고)

### Phase 3 — 콘텐츠 확장 1 (깊이 추가)
아래 "추가 콘텐츠 제안" 중 우선순위 상위 도입.
- **연구/R&D 트리** (CI/CD, 테스트 자동화 → 부채 영구 감소 등).
- **랜덤 이벤트(선택형):** 바이럴/서버다운/투자제안/해커톤 — 트레이드오프 결정.
- **도전과제(Achievements):** 영구 소소 보너스 + 수집 욕구.

### Phase 4 — 메타 & 장기 동기 (Depth)
- **환생 스킬트리:** 경력 포인트로 영구 노드 해금(현재는 단일 +2%만).
- **투자 라운드(Seed→Series A→…):** 마일스톤 대형 현금/배율, 지분 희석 같은 긴장.
- **명성/리뷰:** 유저 획득 배율 자원.

### Phase 5 — 폴리시 & 출시 품질
`motion-ui` / `rules/game/juice.md` / `frontend-design-direction` / `browser-qa` / `click-path-audit`.
- 게임필: 구매/마일스톤/환생 피드백 단계화, 파티클(캔버스 풀링), `prefers-reduced-motion`.
- 사운드(뮤트 토글, 지연로딩 훅).
- 반응형/모바일(현 CSS는 780px 브레이크 보유), 접근성(키보드/대비/ARIA).
- 성능: 라이브 카운터 리렌더 감사, 60fps 틱 유지.
- 정적 빌드 산출 → `index.html` 로컬 더블클릭 또는 `npm run preview`로 실행.

## 추가 콘텐츠 제안 (우선순위)

> 현재 콘텐츠가 얇은 부분을 채우는 아이디어. P0=다음, P1=중기, P2=장기.

| # | 콘텐츠 | 설명 | 새 시스템 | 우선 |
|---|---|---|---|---|
| 1 | **업그레이드 트리** | 클릭강화 / dev×2 / 부채감소 / 오프라인효율 | 영구 구매 | P0 |
| 2 | **연구(R&D) 트리** | CI/CD(부채↓), 테스트자동화(버그↓), 캐싱(생산↑) — LoC/₩로 영구 해금 | 테크트리 | P0 |
| 3 | **랜덤 이벤트(선택형)** | 🔥바이럴(유저 급증) / 💥서버다운(₩손실 vs 부채증가 선택) / 💰투자제안 / ⚡해커톤 | 이벤트+의사결정 | P0 |
| 4 | **도전과제** | "첫 1k LoC", "버그 100회 생존", "무부채 환생" → 영구 소보너스 | 업적 | P0 |
| 5 | **개발자 티어 확장** | 인턴🐣 / 데브옵스🛠️ / 데이터사이언티스트📊 / CTO👔 / 양자컴퓨팅⚛️ | 기존 DEVS 확장 | P1 |
| 6 | **기술 스택 선택** | 언어/프레임워크 선택, 각자 트레이드오프(빠른생산↔높은부채) | 빌드 선택 | P1 |
| 7 | **인시던트 대응** | 부채 폭주 시 장애 발생 → 짧은 대응(클릭연타/선택)으로 완화 — 현 버그연출을 실제 미니게임으로 | 미니게임 | P1 |
| 8 | **환생 스킬트리** | 경력 포인트 분배(생산/부채/오프라인/클릭 계열) | 메타 진행 | P1 |
| 9 | **투자 라운드** | Seed→A→B→IPO 마일스톤, 대형 배율 + 지분 희석 긴장 | 마일스톤 | P2 |
| 10 | **명성/리뷰** | 별점·리뷰가 유저 획득 배율 — 품질 vs 속도 긴장 강화 | 신자원 | P2 |
| 11 | **멀티 프로덕트** | 2번째 제품 라인 해금 → 병렬 경제 | 후기 레이어 | P2 |
| 12 | **이스터에그/밈** | 금요일 배포 금지 이벤트, 스택오버플로 복붙 버프, 콘솔 치트 | 플레이버 | P2 |

### 테마 결합 제안 (장르 신선도 ↑)
- **기술부채↔속도** 축을 모든 신콘텐츠의 공통 트레이드오프로: 빠른 선택은 부채를, 느린/올바른 선택은 시간을 쓴다.
- **연구 트리**가 부채 시스템의 "출구"를 제공해 후반 리팩토링 반복의 단조로움을 완화.
- **랜덤 이벤트 + 인시던트**로 방치 중에도 능동 개입 순간을 심어 리텐션 ↑.
